# Правила разработки через Codex

## Общее

Ты работаешь в проекте X10 Life RPG.

Цель проекта — веб-игра в жанре Life RPG, где пользователь прокачивает персонажа через выполнение реальных жизненных задач.

Backend уже существует и написан на Rust. Frontend должен использовать существующее API из `src/api/routes.rs`.

## Стек frontend

- React
- TypeScript
- Vite
- React Router
- TanStack Query
- Zustand
- Tailwind CSS
- Phaser 3

## Главные правила

- Не переписывать backend без отдельной задачи.
- Не придумывать endpoints, если они уже есть в `api-reference.md`.
- Не хранить server state в Zustand.
- Для запросов использовать TanStack Query.
- Для локального UI/game state использовать Zustand.
- Бизнес-логику не класть в Phaser.
- Phaser должен быть только визуальным игровым слоем.
- Каждый prompt должен заканчиваться рабочей сборкой.

## Перед завершением каждого prompt

- `npm run build` проходит.
- Линтер проходит, если он настроен.
- TypeScript не содержит ошибок.
- Нет случайных `any`.
- Нет мёртвого кода.
- Обновлён `PROGRESS.md`, если задача завершена.
