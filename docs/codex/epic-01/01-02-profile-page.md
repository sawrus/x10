## Фича

Профиль персонажа

## Контекст

Игрок должен видеть и редактировать персонажа.

## Задача для Codex

Создай страницу Character/Profile с карточкой профиля, формой редактирования основных полей и возможностью
повторно открыть уже созданного героя после refresh.

## Используемое API

- `GET /api/v2/profiles`
- `GET /api/v2/profiles/{profile_id}`
- `PATCH /api/v2/profiles/{profile_id}`

## Последний рабочий инкремент

- Character route сохраняет текущий `profileId` между refresh через Zustand persist
- roster над формой/карточкой показывает сохранённых героев с `full_name`, `occupation` и датой создания или последнего изменения
- при открытой карточке можно выбрать другого героя из roster и открыть именно его, без возврата к уже открытому профилю
- `GET /api/v2/profiles` теперь отдаёт `created_at` и `updated_at`, чтобы UI мог честно показывать дату в списке
- `make e2e-test` покрывает создание героя, refresh, повторное открытие и переключение между сохранёнными героями
